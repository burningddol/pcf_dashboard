# Patterns

복수 컴포넌트가 함께 동작하는 화면 레벨 패턴.

---

## 이중 렌즈 (Dual Lens)
**문제** — 같은 활동 데이터를 두 멘탈 모델에서 봐야 함:
- **GHG Scope** — 보고서 작성용 (CDP, 공시)
- **PCF Lifecycle** — 의사결정용 (어디를 줄일까)

**해결**
- 메인 시각화 카드 헤더에 segmented toggle 1개로 전환
- Scope 뷰: stacked bar (시계열) + 우측 12개월 합계 breakdown
- Lifecycle 뷰: Sankey (활동 → 단계 → 총합)
- 토글 상태는 URL/state에 보존, 사용자별 마지막 선택 기억

**규칙**
- 두 뷰는 **같은 데이터셋, 다른 분류축**. 합계는 항상 일치해야 함.
- Scope 색(s1/s2/s3)은 Scope 뷰에서만, Lifecycle 색(raw/mfg/dist/use/eol)은 Lifecycle 뷰에서만.
- 보조 시각화(Treemap, Waterfall)는 옵션으로만 — 기본은 Sankey.

## 위→아래 위계 (경영자 → 실무자)
**문제** — 같은 화면을 두 사용자가 사용. 경영자는 요약, 실무자는 디테일.

**해결 — 하나의 페이지에 4단 위계**
1. **상단 KPI 5장** — 경영자의 답이 여기서 끝남
2. **메인 시각화 카드** — 둘 다 사용. 의사결정 토론의 근거.
3. **활동 데이터 테이블** — 실무자 진입점
4. **사이드패널 계산 풀이** — 실무자의 깊이

**규칙**
- 경영자 뷰에서는 3단을 축소(요약 카드 1개로 대체), 4단은 숨김.
- 실무자 뷰는 사이드패널 기본 열림.
- 페르소나 토글이 있으면 둘 다, 없으면 항상 풀 위계 노출.

## 모든 숫자는 자기 자신을 설명한다 (Number Self-Audit)
**원칙** — 사용자는 어느 셀이든 클릭하면 그 값이 어떻게 나왔는지 역추적할 수 있어야 한다.

**구현**
- 활동 테이블 행 클릭 → 사이드패널 (계산식 + 계수 + 감사 추적)
- KPI 카드 → 클릭 시 해당 메트릭의 정의 + 집계 쿼리 표시 (popover)
- 차트의 한 막대/한 링크 클릭 → 그 부분에 기여한 활동 데이터로 필터된 테이블

**금지**
- 출처 없는 집계 숫자
- "기타" 카테고리 (집계 후 모르겠음 = 데이터 품질 문제이므로 명시적 처리)

## Validation-Block Import
**원칙** — 잘못된 임포트는 분기 보고를 망친다. 사후 정정보다 사전 차단이 훨씬 싸다.

**4단 스테퍼**
1. 파일 업로드 → done
2. 컬럼 매핑 → done
3. 검증·계수 매칭 → **active** (여기서 차단)
4. 확정 임포트 → wait

**검증 결과 표시**
- 페이지 상단 빨강 배너: "**N건의 검증 오류로 임포트가 차단되었습니다**" + 무엇/왜/어떻게 3단
- 미리보기 테이블에서 오류 행은 `neg-soft` 배경
- 오류 행 아래 펼쳐진 설명 + "인라인 수정" / "+ 계수 등록" 버튼
- 페이지 하단: `[N건 임포트 (오류 M건 차단됨)]` — 오류가 있으면 disabled

**오류 유형별 처리**
| 유형 | 표시 | 1차 액션 |
|---|---|---|
| 단위 불일치 | 단위 셀에 `neg` 색 + 600 | 인라인 수정 |
| 계수 미매칭 | 계수 셀 "— 미매칭" `neg` 색 | + 계수 등록 |
| 음수 값 | 활동량 셀에 `neg` 색 + 600 | 인라인 수정 |

## Versioned Reference Data (계수 버전 관리)
**문제** — 배출계수는 연도·고시 버전마다 달라진다. 과거 데이터는 그 시점의 계수로 보존되어야 함.

**해결**
- 각 계수는 `validFrom` 명시
- 새 버전 등록 시 기존 활성 버전의 `validTo`를 신규 `validFrom - 1일`로 자동 설정 → 자동 archive
- 과거 활동 데이터의 계산 결과는 그대로 보존
- 재계산은 별도 명시적 "재계산 작업"으로만 트리거

**UI**
- 계수 카탈로그 테이블 + 우측 sticky 패널에 선택된 계수의 버전 타임라인
- 타임라인: 도트 + 라인. 현행은 `accent` 채워진 도트, 과거는 빈 도트
- 각 버전 항목: 버전명 + 값 + 변화율 pill + 유효기간 + 변경 사유 + 변경자
- "+ 새 버전" 버튼은 패널 헤더에

## 한국어 + 영문 도메인 용어 병기
**규칙** — 도메인 핵심 용어는 한국어가 주, 영문이 괄호 보조.

**예시**
- `원료(Raw Material)` `제조(Manufacturing)` `유통(Distribution)`
- `구매전력(Scope 2)` `직접배출(Scope 1)` `가치사슬(Scope 3)`
- `배출계수(Emission Factor)` `생애주기(Lifecycle)`
- 약어는 한국어 후 — `생애주기 평가(LCA)`, `제품 탄소 발자국(PCF)`

**금지**
- 영문 라벨만 단독으로 (실무자 진입 장벽)
- 한국어 라벨에 영문 미포함 (감사·국제 보고 시 모호)

## Number Formatting (필수)
- 모든 숫자: 천 단위 구분 (`1,234,567`)
- 소수점: 배출량 1~2자리, 계수 4자리, 비중 1자리
- 단위는 별도 span (`var(--fg-3)`), 숫자보다 작게
- 부호 있는 변동: `−` (U+2212) 또는 `+` 사용. ASCII `-` 금지
- 백만 단위 환산 시: `20.48 백만 ₩` (혼합 표기 허용)
- tCO₂e / kgCO₂e: 아래첨자 `₂` 사용 (U+2082)

## State Coverage Checklist
모든 데이터 의존 컴포넌트는 4가지 상태 디자인을 가져야 한다:
- [ ] **Empty** — 데이터 없음. 아이콘 + 설명 + 다음 액션 2개
- [ ] **Loading** — 스켈레톤 (콘텐츠 가짜로 채우지 말 것)
- [ ] **Error** — 시스템 오류. 무엇/왜/어떻게 3단 + 재시도
- [ ] **Partial** — 부분 데이터. warn pill + 추정 범위 + 누락 보기 액션

## 모바일 반응형
- 우선순위: 데스크톱 (1440+) → 태블릿 (768+)
- 모바일은 핵심 뷰만 — KPI 1열 스택, 차트는 가로 스크롤, 테이블은 카드 리스트로 변환
- 사이드패널은 모바일에서 풀스크린 모달
- 좌측 네비 드로어는 햄버거로

## Persona Mode (Optional)
- 토글 기본값: `자동` (사용자 역할 기반)
- `경영자` — KPI + 요약 차트만, 테이블은 "활동 데이터로 이동" 버튼으로 대체
- `실무자` — 풀 위계 + 사이드패널 기본 열림
- 토글 위치: 상단 우측 (search/bell/cog 좌측)
