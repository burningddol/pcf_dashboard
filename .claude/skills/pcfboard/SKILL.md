# PCFBoard Design System

탄소 회계(Carbon Accounting) SaaS를 위한 데이터 밀도 높은 분석 도구 디자인 시스템. Stripe·Mixpanel 톤을 기반으로, 한국 제조사 시나리오(KEPCO, KEITI, GLEC 계수)를 가정하여 만들어졌습니다.

## 정체성
- **밀도 높은 분석 도구**. 1px 헤어라인, tabular figures, 정보 위계가 명확한 카드 구조.
- **Light-first**, dark mode optional.
- **Sustainability는 데이터 색상에서만**. 베이스 UI는 거의 무채색이며, sustainability 톤은 Scope 차트 팔레트(딥 틸 → 틸 → 라임)에서만 드러납니다.
- **숫자가 주인공**. 모든 숫자는 단위와 자리수를 가지며, IBM Plex Mono의 tabular figures로 정렬됩니다.

## 사용 방법
1. `tokens.json` / `tokens.ts` — 모든 디자인 결정의 단일 진실 공급원.
2. `tokens.css` — 즉시 사용 가능한 CSS 변수 + 유틸리티 클래스 (`.pcf` 스코프).
3. `components.md` — 핵심 컴포넌트의 시각·인터랙션 사양.
4. `patterns.md` — 두 개 이상의 컴포넌트가 함께 동작하는 화면 레벨 패턴.

## 디자인 원칙 (충돌 시 우선순위)
1. **정확성** — 단위·자리수·계수 버전이 명시적으로 표시.
2. **추적 가능성** — 모든 결과값은 입력 데이터까지 역추적 가능.
3. **위계** — 경영자는 위에서, 실무자는 아래에서 답을 얻음.
4. **미니멀리즘** — 차트 한 종류로 두 가지 의미를 보여줄 수 있다면 그렇게.

## 절대 하지 말 것
- 지구 일러스트, 잎사귀 아이콘 등의 sustainability 클리셰
- 수치 없는 장식적 게이지·도넛
- 베이스 UI에 saturation > 0.02 의 색
- 단위 없는 숫자
- 활동 데이터의 계산 과정을 숨기는 것 (모든 숫자는 자기 자신을 설명할 수 있어야 함)

## 타입 시스템
- 본문: Pretendard / IBM Plex Sans KR
- 숫자: IBM Plex Mono (tabular)
- 한국어 라벨 + 영문 도메인 용어 병기 패턴 — 예: `원료(Raw Material)`, `구매전력(Scope 2)`

## Scope 팔레트 (도메인 핵심)
| Scope | 의미 | 색 |
|---|---|---|
| 1 | 직접배출 (회사가 직접 태우는 연료) | `#0F4C5C` 딥 틸 |
| 2 | 간접배출 (구매한 전기·스팀) | `#2A9D8F` 틸 |
| 3 | 가치사슬 배출 (원료·물류) | `#A7C957` 라임 |

이 세 색은 차트·뱃지·계수 분류에서 일관되게 사용되며, 다른 용도로는 절대 사용하지 않습니다.
